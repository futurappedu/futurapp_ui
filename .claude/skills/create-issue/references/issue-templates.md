# Issue Templates Reference

## Bug Report Template

```markdown
## Description

[Clear, concise description of the bug]

## Steps to Reproduce

1. Navigate to [page/feature]
2. Perform [action]
3. Observe [unexpected result]

## Expected Behavior

[What should happen]

## Actual Behavior

[What actually happens]

## Environment

- Browser: Chrome / Firefox / Safari
- OS: macOS / Windows / Linux

## Screenshots / Error Logs

[Attach if available]

## Additional Context

[Any other relevant information]
```

## Feature Request Template

```markdown
## Problem Statement

[What pain point or limitation does this address?]

## Proposed Solution

[Describe the desired feature or behavior]

## User Story

As a [student/admin], I want [feature] so that [benefit/outcome].

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Design Considerations

[UI mockups, UX flow, or technical approach if relevant]

## Alternatives Considered

[Other approaches and why they were not chosen]

## Dependencies

[Related issues, external services, or prerequisites]
```

## Technical Debt Template

```markdown
## Current State

[What technical debt exists?]

## Proposed Improvement

[How should it be refactored?]

## Motivation

- [ ] Code maintainability
- [ ] Performance
- [ ] Security
- [ ] Developer experience
- [ ] Test coverage

## Impact Assessment

- Files affected: [list key files]
- Risk level: Low / Medium / High
- Estimated scope: Small / Medium / Large
```

## Label Quick Reference

### Type

- `bug` — Something isn't working
- `enhancement` — New feature or improvement
- `documentation` — Documentation updates

### Priority

- `High` — Critical, blocking work
- `Medium` — Important but not urgent
- `Low` — Nice to have

### Areas

`tests`, `admin`, `scholarships`, `recommender`, `auth`, `profile`, `ui`, `api`

### Categories

`aptitude-tests`, `personality-test`, `career-recommendations`, `scholarship-search`, `bulk-import`, `performance`, `security`
